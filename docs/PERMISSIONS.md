# Benjamin PMS — Permission Matrix (RBAC)

> 6 บทบาท × ทรัพยากรหลัก. `✓`=ทำได้ทุกตัวในขอบเขต, `✓*`=ทำได้แบบ **act-on-behalf** (กรอก/แก้แทนดีลเลอร์ + บันทึก audit ว่า HQ เป็นคนทำ), `◐`=เฉพาะที่ตนเกี่ยวข้อง/ตามมอบหมาย, `R`=อ่านอย่างเดียว, `–`=ไม่ได้
> **ขอบเขตข้อมูล (Scope):** HQ roles = ทุกดีลเลอร์ · Dealer roles = เฉพาะ `dealerId` ของตน

## 3. กฎพิเศษ (Business Rules)
1. **การอนุมัติใบเสนอราคา** — ถ้า `discountPct > 10%` หรือ `total < ราคาพื้น` → `approvalStatus=PENDING` ต้องให้ HQ_MANAGEMENT/SUPER อนุมัติ; มิฉะนั้น DEALER_ADMIN อนุมัติได้เอง
2. **การมอบหมายลีด** — ลีดใหม่จากเว็บ/LINE เข้า pool กลาง (dealerId=null) → ระบบเสนอดีลเลอร์ตามจังหวัด → HQ หรือ DEALER_ADMIN ในเขตยืนยัน
3. **โอนข้ามดีลเลอร์** — เฉพาะ HQ (หรือ DEALER_ADMIN ต้นทางส่งคำขอ → HQ อนุมัติ)
4. **ราคากลาง** — แก้ไขได้เฉพาะ SUPER/HQ_STAFF; ดีลเลอร์ปรับราคาในใบเสนอราคาได้ภายใน margin ที่กำหนด
5. **ทุก write** → เขียน AuditLog (ใคร/เมื่อไหร่/ก่อน-หลัง)
6. **Act-on-behalf (HQ_MANAGEMENT/SUPER แก้แทนดีลเลอร์)** — เมื่อ HQ กรอก/แก้ข้อมูลของดีลเลอร์ใด ระบบบันทึก AuditLog โดยระบุ `onBehalfOfDealerId` (ดีลเลอร์เจ้าของข้อมูล) + `actedByRole=HQ` เพื่อให้แยกได้ว่า "HQ เป็นคนทำแทน" ไม่ใช่ดีลเลอร์เอง; ข้อมูลที่ถูกแก้ยังผูก `dealerId` ของดีลเลอร์เจ้าของตามเดิม
